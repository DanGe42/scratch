Lendmo

Lendmo is a social lending tracker that makes it simple to track what you have lent to or borrowed from your friends.

Lendmo was my team's Spring 2012 PennApps submission. This web application was written primarily in Ruby on Rails, and it relies on Facebook for authentication. I implemented integration with the Venmo API to allow users to pay borrowers back for item impairment (i.e. the user broke whatever was lent to them). This was the first time I ever worked with Ruby and Rails in a practical setting, and this project was a great way for me to learn Rails and the MVC paradigm.

Clickr

Clickr is a Twitter-based replacement for clickers. Clickers are useful because a professor can use them to poll his or her students when, for example, he or she posts a sample multiple choice question. However, these devices can be expensive for faculty and students, and thus, we wrote this web app. Clickr works by parsing tweets directed at a course Twitter account and charting the results (in realtime).

Clickr was my team's entry into the Fall 2011 PennApps Hackathon. This web application was written primarily with PHP and JavaScript. I primarily contributed to the JavaScript frontend logic; I wrote JS to pull data from the Twitter search API, and then used Highcharts to visualize the data. Using jQuery helped cut down on much of the code clutter and simplified subroutines like AJAX calls and JSON parsing.

Rage-Racer

Rage-Racer is a racing game built as a mobile web app. It allows users to race other smartphone users on a central server. To play, users simply tap their phones to move their characters represented with rage faces (or if they want, they can run their fingers on their phones) and shake their phones to jump hurdles until they reach the finish line.

Rage-Racer was my team's (different team from the one I worked with for Clickr) entry into the Spring 2011 HackNY Hackathon. This mobile web app was written primarily with a node.js backend and a HTML5 canvas frontend. I primarily contributed to the canvas frontend and JavaScript logic that displayed the racetrack, locations of the characters, and the characters' movements in response to run or jump inputs.

JAsteroids

JAsteroids is an Asteroids clone that I built for my CIS 120 final project. The game is written in Java and uses the Swing toolkit. This clone is based off the version of Asteroids that came with Windows XP. There is still quite a bit of work left for me because the asteroids are all squares (could not really figure out how to do collision detection efficiently) and movement is still quite awkward.
